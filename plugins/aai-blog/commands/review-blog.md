# /review-blog - Review Blog Post

Review a technical blog post for quality, accuracy, and engagement.

## Usage

```
/review-blog <file-path>
/review-blog posts/react-hooks-guide.md
```

## Review Criteria

1. **Technical Accuracy**
   - Verify code examples are correct
   - Check facts and claims
   - Ensure best practices are followed

2. **Clarity & Structure**
   - Clear introduction and conclusion
   - Logical flow between sections
   - Appropriate heading hierarchy

3. **Engagement**
   - Hook in the introduction
   - Practical, relatable examples
   - Clear value proposition

4. **Code Quality**
   - Examples are complete and runnable
   - Proper syntax highlighting
   - Comments where helpful

5. **SEO & Discoverability**
   - Title effectiveness
   - Meta description quality
   - Keyword usage

6. **Readability**
   - Sentence length variety
   - Technical jargon explained
   - Appropriate for target audience

## Output

- Overall assessment
- Specific issues with line references
- Suggested improvements
- Strengths to maintain
