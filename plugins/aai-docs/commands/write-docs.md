# /write-docs - Write Documentation

Write comprehensive documentation for your code, features, or systems.

## Usage

```
/write-docs [target]
/write-docs authentication system
/write-docs src/services/payment.ts
/write-docs getting started guide
```

## Workflow

1. **Understand the Target**
   - Analyze code/system to document
   - Identify the audience
   - Determine documentation type

2. **Gather Information**
   - Explore relevant code
   - Identify public APIs
   - Note configuration options

3. **Write Documentation**
   - Clear overview/introduction
   - Step-by-step instructions
   - Code examples
   - API reference if applicable

4. **Add Supporting Content**
   - Prerequisites
   - Troubleshooting section
   - FAQ if relevant

5. **Review & Polish**
   - Verify accuracy
   - Check completeness
   - Improve clarity

## Documentation Types

- **README** - Project overview
- **Guide** - How-to instructions
- **Reference** - API documentation
- **Tutorial** - Step-by-step learning
- **Architecture** - System design docs

## Output

- Complete documentation in markdown
- Code examples
- Diagrams if needed
